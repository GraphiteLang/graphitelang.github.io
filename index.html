GraphiteLang

A C styled, nim based, multi-level, multi-paradigm programming language. Stay tuned for updates!

Core Features

Dynamic Containers
Flexible data structures that can grow or shrink at runtime, like lists and hash maps.
Async Expression
Expressions which evaluate without blocking, crucial for I/O and concurrent tasks.
Deferred Expressions
Expressions that evaluate automatically when the surrounding scope exits, useful for cleanup.

Memory Management

nim-ORC
A modern, compile-time garbage collector designed for performance and determinism.
nim-ARC
Provides automatic memory management with predictable deallocation.
nim-Boehm
A conservative garbage collector for compatibility and ease of use with C FFI.
nim-Go
A concurrent, tri-color mark-and-sweep garbage collector for highly concurrent applications.
Implicit Ownership
Graphite's unique memory model aiming for C-like performance with Rust-like safety, without explicit annotations.

Constructs

Tuples
Fixed-size collections of anonymous fields for structured data organization.
Interfaces
Dataless views which map identity onto tuples by shape and provide behavior.
Structs
Traditional Struct implementation as a Tuple/Interface pair.
Classes
Typical instantiable namespace for object-oriented programming patterns.
Actors
Threaded, memory isolated Classes for safe concurrent programming.

Concurrency

Coroutines
Lightweight, cooperative functions that allow for non-blocking asynchronous programming.
Work Groups
Mechanism for grouping and managing related tasks or threads for efficient parallel processing.
